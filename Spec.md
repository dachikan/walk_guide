# Walk Guide - 視覚障害者向け歩道案内「ロボット大脳」アプリ

## プロジェクト概要

### プロジェクト名
Walk Guide - 視覚障害者のための「ロボット大脳」型歩行支援アプリ

### 用途：視覚障害者の介助者

視覚障害者の方が安全に歩道を歩行できるよう支援する盲導犬の役割を果たすスマホ散歩案内アプリです。
地点間を結ぶルートで案内します。
案内に当たっては、ルート前方を胸部に装着したスマホで数秒ごとに自動撮影し、AI画像解析により障害物・危険物・ランドマークを検出して音声で案内します。

### 基本コンセプト：ロボットOS（ROS）的アーキテクチャの導入
本アプリは単なるスマホアプリではなく、**「スマートフォンというハードウェアに搭載された、自律歩行ロボットの知能（大脳）ユニット」**として設計開発されます。
ロボット工学における「Sense（認知）- Think（判断）- Act（行動）」のサイクルをスマホ内で完結させ、視覚障害者の安全な移動を支援します。

## 開発の最重要前提条件：ロボット的安全設計

### 1. 冗長性と合意形成 (Redundancy & Consensus)
- **単一AIに依存しない**: 命に関わる「前方OK」の判断は、将来的にローカル解析（MediaPipe等）とクラウドAI（Gemini/Claude等）のダブルチェック（合意形成）を必須とします。
- **フェイルセーフ**: センサー（カメラ）や通信に少しでも異常がある場合は、最も保守的な案（「注意してください」「停止してください」）を選択します。

### 2. 反射（小脳）と熟考（大脳）のハイブリッド
- **小脳（ローカル解析）**: デバイス上のMediaPipe等で「人」「車」「段差」をリアルタイムにトラッキング。ミリ秒単位の「反射的」な警告を担います。
- **大脳（クラウドAI）**: 数秒おきの画像解析により、道路の状況やナビゲーション等の高度な「状況判断」を行います。

### 3.AI介助者としての責任自覚

#### 重要な責任認識

- **「前方OK」は視覚障害者の命に関わる重要な判断**
- **少しでも危険の可能性があれば警告を優先する**
- **「安全すぎる」は良いことで、「見落とし」は絶対に避ける**
- **障害物・段差・工事・人通りなど全てを慎重に判断する**
- **不明な場合は「注意して進んでください」などの保守的な案内**

#### 禁止事項

- **楽観的な「前方OK」判定の乱発**
- **「大丈夫だろう」という推測に基づく安全判断**
- **見えにくい場合の安易な「OK」判定**

この前提に基づき、すべてのAIプロンプトは視覚障害者介助者としての慎重さを持って設計される。

## システムアーキテクチャ（ロボット構成）

### 1. Sense（認知ノード）
- **Camera Node**: 10秒間隔の自動撮影およびリアルタイムプレビュー。
- **Voice Recognition Node (STT)**: 利用者からの音声操作コマンドを解析。

### 2. Think（推論判断ノード：WalkingBrain）
- **WalkingBrain クラス**: AIサービス（Gemini, Claude, ChatGPT）を抽象化し、入力を指示に変換する「中央集権型大脳」。
- **海馬（短期記憶）**: 10秒前と現在の画像を比較し、状況の変化（信号の変化、接近する物体）を捉える「ロボットの記憶」。

### 3. Act（行動出力ノード）
- **TTS Node**: 判断結果を自然な日本語音声（音声指示）として出力。
- **Haptic Node（ハートビート）**: 4秒間隔の生存確認バイブレーション。

## 3状態システム (Robot State Machine)

1. **AppState.normal（通常動作中）**
   - 10秒間隔の前方解析サイクルを実行。マイクボタンは**青色**。
2. **AppState.listening（コマンド待機中）**
   - ユーザーの音声入力を受け付け中。マイクボタンは**赤色**。
3. **AppState.processing（処理発話中）**
   - AI解析中またはTTS発話中。重複入力を防ぐためマイクボタンは**赤色**。

## 設定画面（補助者用インターフェース）

### ⭐ 前提条件：設定操作は補助者が行う

**視覚障害者には必ず晴眼者の補助者が同行する前提とする。**

- 設定ボタン（AppBar右上の⚙️アイコン）は**補助者が目で見て操作**するものとする
- 視覚障害者本人が操作することは想定しない
- そのためUI上のアクセシビリティ対応（TalkBack等）は設定画面には不要
- 設定操作中は視覚障害者向けの音声コマンドを無効化し、補助者の操作を妨げない

### 設定ボタンの操作内容

- **配置**: AppBar 右上の⚙️アイコン
- **操作**: タップで**設定メニュー画面**に遷移する

### 設定メニュー画面の構成

設定ボタンを押すと以下の項目を持つ設定メニュー画面が表示される。

```
┌────────────────────────┐
│  ⚙️ 設定               │
├────────────────────────┤
│  🤖 AIサービス選択     │
│  🎤 音声認識調整       │
│  📍 ルート設定（将来）  │
└────────────────────────┘
```

#### 🤖 AIサービス選択

- ラジオボタンで使用AIを選択（Gemini / Claude / ChatGPT）
- 各AIサービスの特徴説明付き
- 選択内容は SharedPreferences に保存され次回起動時も維持

#### 🎤 音声認識調整

- **初回起動時だけでなく、設定メニューからいつでも再調整可能**
- 音量しきい値スライダー（0.01 〜 0.50）
- 無音検出時間ドロップダウン（1〜5秒）
- マイクテスト・コマンドテスト実行ボタン付き
- 設定内容は SharedPreferences に保存

#### 📍 ルート設定（将来実装予定）

- ルートの選択（選択中のルートをAIプロンプトに渡す）
- 地点追加・削除・並び替え機能
